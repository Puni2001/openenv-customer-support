🎮 Task Levels
Six progressive difficulty levels — from simple classification to multi-agent pipelines under chaos conditions.
Easy — Ticket Triage
Beginner
Classify incoming tickets into the correct category: technical, billing, account, feature_request, or complaint.
+0.40 correct | +0.10 valid-wrong | -0.30 invalid
Medium — Prioritization
Intermediate
Classify AND set the correct priority (low/medium/high/urgent) based on sentiment, SLA deadline, and VIP status.
+0.35 × SLA urgency multiplier | VIP awareness bonus
Hard — Full Resolution
Advanced
Resolve tickets using the knowledge base OR escalate when appropriate. Empathy, SLA compliance, and repeat-contact handling all scored.
+0.50 KB quality | +0.15 empathy | +0.10 SLA ok | -0.20 breach
Chaos — Ticket Storm
Expert
8 tickets at once. Some VIP, some SLAs already breached, some repeat contacts. Tests robustness under real enterprise pressure.
Urgency-weighted scoring | VIP 2× weight multiplier
Multi-Agent Triage
Multi-Agent
Agent 1 in a two-agent pipeline. Categorize the ticket and route it to the correct specialist team.
Routing accuracy | Correct category + escalation path
Multi-Agent Resolver
Multi-Agent
Agent 2 in the pipeline. Receives pre-triaged tickets and resolves or escalates with full context from Agent 1.
Resolution quality | Escalation correctness | SLA compliance
⚡ Reward Model
Dense shaped rewards — the agent gets a signal at every step, not just at the end. SLA urgency multiplies the priority reward in real time.
ActionConditionReward
categorizeCorrect category+0.40 (+0.05 reasoning bonus)
categorizeValid but wrong category+0.10
categorizeInvalid value-0.30
prioritizeCorrect priority+0.35 × SLA urgency (1.0 / 1.2 / 1.5×)
prioritizeOff by one level+0.10
prioritizeUnder-prioritize urgent-0.25 + -0.30 extra
prioritizeVIP customer, high/urgent+0.10 VIP bonus
resolveKB-aligned resolution+0.50 × quality score
resolveEmpathy words for negative sentiment+0.15 empathy bonus
resolveSLA not breached+0.10
resolveSLA breached-0.20
escalateCorrect escalation decision+0.35 (+0.05 reasoning)
escalateUnnecessary escalation-0.40
anyStep cost (efficiency)-0.01 per step
🛡️ Anti-Reward Hacking
RL agents are notorious for "specification gaming" (optimizing the proxy metric instead of the real goal). This environment actively detects and penalizes shortcuts.
🚨 Exploit: Priority Spam
Agent marks every ticket as URGENT to maximize the SLA priority multiplier, ignoring actual sentiment.
Protection: -0.40 hack penalty for marking Low/Medium tickets as Urgent.
🚨 Exploit: Fake Empathy
Agent spams "sorry" to get the +0.15 empathy bonus without actually providing KB steps to resolve the issue.
Protection: -0.20 penalty if empathy words are used but resolution quality score is < 40%.
⚙ Setup Guide
Get the environment running locally in under 5 minutes.
1
Clone and install
Clone the repo and install dependencies into a virtual environment.
# Clone from HuggingFace Space
git clone https://huggingface.co/spaces/punith2001/ai-support-envoy
cd ai-support-envoy
python -m venv venv && source venv/bin/activate
pip install -r requirements.txt
2
Add your HuggingFace token
Create a free token at huggingface.co/settings/tokens. The environment uses the HF Router for free LLM inference.
# .env file
HF_TOKEN=hf_your_token_here
API_BASE_URL=https://router.huggingface.co/v1
MODEL_NAME=Qwen/Qwen2.5-72B-Instruct
3
Validate the environment
Confirm OpenEnv compliance before running anything.
openenv validate .
# Expected: [OK] : Ready for multi-mode deployment
4
Run the inference baseline
Runs the LLM agent across easy, medium, and hard tasks. Outputs [START] / [STEP] / [END] logs.
python inference.py
# [START] task=easy env=customer-support-env model=Qwen/Qwen2.5-72B-Instruct
# [STEP] step=1 action=categorize(technical) reward=0.4500 done=false error=null
# [END] success=true steps=3 score=0.9900 rewards=0.4500,0.4500,0.4500
5
Use the environment in Python
Plug it into your own agent or training loop.
from src.customer_support_env import CustomerSupportEnv, Action

env = CustomerSupportEnv(task_level="hard")
obs = env.reset()

action = Action(
    action_type="resolve",
    value="Apologize for the frustration. Diagnosed error logs, restarted service.",
    reasoning="Negative sentiment + technical issue"
)
obs, reward, done, info = env.step(action)
# reward=0.74, breakdown={'resolution_quality':0.5,'empathy_bonus':0.15,...}
6
Train with GRPO (on Colab or with compute credits)
Run the full curriculum training. Open train_colab.ipynb on Colab with a T4 GPU.
# Full curriculum: easy → medium → hard
python train.py --model Qwen/Qwen2.5-0.5B-Instruct --curriculum easy,medium,hard
